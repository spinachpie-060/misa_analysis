# GitHub アカウントの作り方（学生向けガイド）

---

## **1. GitHub とは？**

GitHub（ギットハブ）は、プログラムやレポートなどのファイルを

インターネット上で管理・共有できるサービスです。

- 自分のコードやレポートを **保存** できる
- バージョン管理（いつ、どこを変更したか）を **記録** できる
- 共同作業（グループワーク）が **やりやすくなる**

この実習では、GitHub アカウントを使って

**クラウド上の Linux 環境（GitHub Codespaces）** にログインします。

---

## **2. 事前に用意しておくもの**

1. **メールアドレス**
    - できれば **大学のメールアドレス** を使ってください。
    - 例：xxxxx@eis.hokudai.ac.jp など
2. **インターネットが使える環境**
    - PC でもタブレットでも構いませんが、PC推奨です。
3. **Webブラウザ**
    - Google Chrome（おすすめ）
    - Microsoft Edge
    - Mozilla Firefox など

---

## **3. GitHub のトップページを開く**

1. Webブラウザを開きます。
2. アドレスバーに次のURLを入力します：

```
https://github.com
```

1. GitHub のトップページが開きます。

---

## **4. サインアップ（新規登録）を開始する**

1. 画面右上の **「Sign up」**（サインアップ）ボタンをクリックします。
    
    （または、トップページ中央あたりに「Sign up for GitHub」と出ている場合もあります）
    
2. 「Create your account」（アカウント作成）画面が表示されます。

---

## **5. 必要な情報を入力する**

画面の指示にしたがって、次の情報を順番に入力します。

### **5-1. メールアドレスを入力**

- Email address の欄に、自分のメールアドレスを入力します。
    - 例：taro.yamada@eis.hokudai.ac.jp
- 入力したら、下のボタン（Continue など）を押します。

### **5-2. パスワードを設定**

- Password の欄に、**自分で決めたパスワード** を入力します。
- 条件（8文字以上・英数字・記号など）が表示されるので、それを満たすようにしてください。
- パスワードは **忘れないように、どこか安全な場所にメモ** しておきましょう。
- 入力できたら、Continue を押します。

### **5-3. ユーザ名（ユーザー名）を決める**

- Username の欄に、自分の GitHub 上での名前を入力します。
    - 例：taro-yamada, taro123, yamada-lab-2025 など
- 半角英数字・ハイフン（-）が使えます（日本語はNG）。
- 後から変更できますが、あまり長すぎず、他人と区別できる名前がよいです。
- 入力した名前が他の人と重なっていると、エラーになります。その場合は少し変えて再入力します。
- 入力できたら、Continue を押します。

### **5-4. メール配信の設定**

- 「Do you want to receive product updates and announcements via email?」などと聞かれることがあります。
- 実習目的であれば、どちらでも構いません。
    - 迷惑メールを減らしたい → No（受け取らない）
    - GitHub からの情報も見たい → Yes（受け取る）
- 選択したら、Continue を押します。

---

## **6. ロボットでないことの確認（CAPTCHA）**

- 画面に「Verify your account」や「Solve this puzzle」などと表示されることがあります。
- 指示にしたがって、簡単なパズルや画像選択を行います。
    - 例：「同じ形を合わせる」「指定された画像をクリックする」など
- 終わったら Create account または Next を押します。

---

## **7. メールアドレスの確認（Email Verification）**

1. 先ほど登録したメールアドレスに、GitHub から確認メールが届きます。
    - 件名の一例：“Verify your email address” など
2. メールを開き、本文中の **「Verify email」** ボタンやリンクをクリックします。
3. ブラウザが開き、GitHub の画面で **アカウントの確認が完了** します。

※ この「メール確認」をしないと、GitHub の一部機能（特に Codespaces やリポジトリ操作）がうまく使えないことがあります。

---

## **8. アカウント作成後の初期設定（簡易）**

メール確認が終わると、GitHub の画面が表示されます。

- 「What do you plan to use GitHub for?」（何に使う予定？）
    
    → 実習では何を選んでも問題ありません。
    
    → わからなければ Skip personalization や Skip があればスキップして構いません。
    
- テーマ選択（Light / Dark）
    
    → 好きなものを選んでください。
    

これで **GitHub アカウントの作成は完了** です。

---

## **9. 正しくログインできるか確認する**

1. もう一度 https://github.com にアクセスします。
2. 右上のメニューに、自分のアイコンと **ユーザー名** が表示されていればOKです。
3. もしログアウトされている場合は、
    - 右上の **「Sign in」** をクリック
    - 登録した **メールアドレスまたはユーザ名** と **パスワード** でログインします。

---

## **10. よくあるトラブルと対処**

### **Q1. 確認メールが届きません**

- 迷惑メールフォルダ（Spam）に入っていないか確認してください。
- メールアドレスを間違えて登録していないか確認してください。
- それでも届かない場合：
    - GitHub にログイン → 右上のアイコン → Settings → Emails
        
        から、Resend verification email（確認メールを再送）を押すことができます。
        

### **Q2. パスワードを忘れてしまいました**

- GitHubのログイン画面で Forgot password? をクリックします。
- 登録したメールアドレスを入力すると、パスワードリセット用のメールが届きます。

### **Q3. ユーザ名が何度やってもエラーになります**

- すでに誰かが使っているユーザ名は登録できません。
- 後ろに数字をつける・ハイフンを入れるなどして、少し変えてみましょう。
    - 例：taro-yamada, taro-yamada-2025, tyamada-student など
