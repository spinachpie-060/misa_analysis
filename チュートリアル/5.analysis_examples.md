# misa_like_normalized_filtered.v2.py の出力ファイル（TSVファイル）の分析方法

---
# 1. TSVファイルの確認

ワイルドカートを使って、作業ディレクトリ内のTSVファイルを全て確認したい場合

```
ls *.tsv

```

特定のTSVファイルの冒頭10行を、確認したい場合には、コマンド **head** をつかう！

```
head [TSVファイル名]

```

TSVファイルの内容を、スクロールして、参照したい場合には
コマンド **less** をつかう！

```
less [TSVファイル名]

```

終了したい場合は　"q" を叩く！


---
# 2. 検出された SSR の数を調べる

出力されたTSVファイルの "行数-1" が検出された SSR の数と一致します！
従って、コマンド **wc** を使って、TSVファイルの行数を出力させよう！

例

```
wc -l [TSVファイル名]

```

"-l" は、行数を出力させるためのオプションです。

注意：1行目はヘッダー

実際の SSR 数

行数 - 1

---

# 3. モチーフ頻度（どの配列パターンが多いか？）

SSRの単位となる塩基配列（モチーフ）のうち、どれが最も多く出現するかを集計します。

```bash
cut -f4 [TSVファイル名] | tail -n +2 | sort | uniq -c | sort -nr | head
```

* **コマンドの解説**:
    * `cut -f4`: 4列目（Motif列）だけを取り出します。
    * `tail -n +2`: 1行目のヘッダー（Motifという文字）を除外します。
    * `sort | uniq -c`: 同じモチーフを並べ替えてまとめ、その数を数えます（`-c` は count の略）。
    * `sort -nr`: 数が多い順（数値降順）に並べ替えます。
* **ここから分かること**:
    * 「AT」や「AAT」など、そのゲノムで特に偏って存在する繰り返し配列がわかります。
    * **考察のヒント**: 植物では `AT` 豊富、ヒトでは `AC` 豊富など、生物種による特徴が出ることがあります。

# 4. 繰り返し回数の分布（どれくらい長く連続しているか？）

SSRが何回繰り返されているかの分布を確認します。

```bash
cut -f5 [TSVファイル名] | tail -n +2 | sort -n | uniq -c
```
* **出力結果の見方**:
左側の数字：検出された個数
      右側の数字：繰り返し回数（Repeats）
      
      11 5  => 5回繰り返しのSSRが 11箇所
      32 6  => 6回繰り返しのSSRが 32箇所

* **コマンドの解説**:
    * `cut -f5`: 5列目（Repeats列）を取り出します。
    * `sort -n`: 繰り返し回数を数字として小さい順に並べます。
* **ここから分かること**:
    * 一般的に、繰り返し回数が少ないものほど多く、回数が多いものほど稀（レア）になります。
    * **考察のヒント**: 異常に長い繰り返し（例：30回以上など）が見つかった場合、それはゲノムの中で不安定な領域や、特殊な機能を持つ領域かもしれません。

# 5. 繰り返し回数が多いSSRの抽出

特に長いSSRだけを抜き出して確認します。ここでは例として「10回以上」を抽出します。

```bash
awk '$5 >= 10' [TSVファイル名]
```

* **コマンドの解説**:
    * `awk`: 条件に合う行だけを表示する強力なツールです。
    * `$5 >= 10`: 「5列目の値が10以上のとき」という条件を指定しています。
* **ここから分かること**:
    * 個体間で長さが変わりやすい「多型性の高いマーカー候補」を見つけることができます。

# 6. SSRの位置分布（ゲノムのどこに固まっているか？）

ゲノム上の位置（開始座標）を10kb（10,000塩基）ごとの区画に区切って、どこにSSRが密集しているかを調べます。

```bash
cut -f2 [TSVファイル名] | tail -n +2 | awk '{print int($1/10000)}' | sort | uniq -c
```
* **出力結果の見方**:

出力された2列の数字は、以下のような意味を持っています。

左側の数字： その区画の中に見つかった SSRの個数

右側の数字： 配列の先頭から数えた 区画番号（0〜99）

3 = 30,000 〜 39,999 bp 付近

10 = 100,000 〜 109,999 bp 付近

51 = 510,000 〜 519,999 bp 付近

例： 7 37 という行は、「370,000塩基目あたりの10kbの範囲に、SSRが7個固まって存在している」ことを示します。


* **コマンドの解説**:
    * `awk '{print int($1/10000)}'`: 開始位置を10,000で割り、小数点を切り捨てて「区画番号」を作ります。
    * `uniq -c`: 各区画にいくつSSRがあるかをカウントします。

* **ここから分かること**:
    * SSRがゲノム全体に均等にあるのか、それとも特定の場所に塊（クラスター）として存在するのかが見えてきます。
    * **考察のヒント**: セントロメア付近や遺伝子の少ない領域に偏っている場合があります。



# 7. 複数ファイルの比較（一括処理）

複数のサンプル（TSVファイル）がある場合、それぞれの合計SSR数を一気に表示させます。

```bash
for f in *.tsv; do
  echo "File: $f"
  tail -n +2 "$f" | wc -l
done
```

* **コマンドの解説**:
    * `for f in *.tsv`: ディレクトリ内の全てのTSVファイルを1つずつ変数 `f` に代入してループします。
* **ここから分かること**:
    * サンプル間でのSSR密度の違いを素早く比較できます。
