# 🐧 Linux入門：バイオインフォマティクスへの第一歩

## **1. OS（カーネル）とはなにか？**

コンピュータの中枢には「**カーネル（Kernel）**」という**OSの心臓部**があります。カーネルは、CPUやメモリなどハードウェアを直接制御する役割を担っています。

現代の主要なOSは、その「心臓部」の設計思想によって以下のように分類されます。

| OS名 | 心臓部（カーネル） | 特徴 |
| --- | --- | --- |
| **Windows** | **Windows NT** | Microsoft独自の設計。個人用PCのシェアNO.1。 |
| **macOS** | **XNU (UNIX系)** | UNIXという伝統的なOSをベースに、Appleが美しく進化させたもの。 |
| **Linux** | **Linux** | UNIXをモデルに有志が開発した自由なOS。スパコンやサーバーで圧倒的シェア。 |

---

## **2. なぜバイオインフォマティクスで Linux を使うのか？**

DNAやRNAの巨大な配列データを扱うバイオの世界では、WindowsではなくLinuxが標準として選ばれます。

1. **解析ツールのほとんどが Linux 対応**: BLASTやBWAなど、主要ツールの多くはLinux環境を前提に開発されています。
2. **自動化のしやすさ**: 何百ものサンプルを「一括で処理する」コマンド操作が得意です。
3. **世界標準のインフラ**: 大学のスパコンやクラウド（AWSなど）は、ほぼすべてLinuxで動いています。

---

## **3. シェル（Shell）とはなにか？**

カーネル（心臓部）は直接触ることが難しいため、その**外側にある操作用の窓口**として作られたのが「**シェル（Shell）**」です。

- **語源**: カーネルを包み、外から守る存在 ＝ **貝殻（Shell）** にたとえて名付けられました。
- **役割**: 私たちがターミナルで入力したコマンドを解釈し、カーネルに伝えて実行させます。
- **種類**: 最も一般的な **`bash`** や、高機能な `zsh` などがあります。

## **4. 初心者向け Linux コマンド入門**

### **🧭 パス（場所）の指定方法**

Linuxでは、ファイルやディレクトリ（フォルダ）の場所を「パス」で指定します。

- **絶対パス**: `/`（ルート）から始まる完全な道順。例: `/home/user/data/`
- **相対パス**: 今いる場所（`.`）からの道順。例: `./data/`
- **特殊記号**: `~`（ホーム）、`..`（一つ上の階層）

### **🛠 最初に覚えるべき基本コマンド10選**

| コマンド | 内容 | 使い方（例） |
| --- | --- | --- |
| **`ls`** | ファイル一覧を表示 | `ls -l`（詳細表示） |
| **`cd`** | ディレクトリを移動 | `cd ..`（一つ上へ） |
| **`pwd`** | 現在の場所を確認 | `pwd` |
| **`cat`** | 中身をそのまま表示 | `cat file.txt` |
| **`less`** | スクロールして閲覧 | `less large_file.txt` (終了は `q`) |
| **`mkdir`** | フォルダを作る | `mkdir result` |
| **`touch`** | 空のファイルを作る | `touch sample.txt` |
| **`cp`** | コピーする | `cp old.txt new.txt` |
| **`mv`** | 移動・名前変更 | `mv old.txt new_name.txt` |
| **`rm`** | 削除（注意！） | `rm -r folder`（フォルダごと削除） |

### **💡 便利な小技**

- **Tabキー**: 入力中のファイル名を自動で補完してくれます。
- **↑ / ↓ キー**: 過去に打ったコマンドの履歴を呼び出せます。
- **Ctrl + C**: 実行中の処理を途中で強制停止します。

---

📘 **まとめ** Linuxは「慣れれば最強の味方」です。まずは `ls` と `cd` から始めて、この“貝殻”を自由に開けるようになりましょう！
