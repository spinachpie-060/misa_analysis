# 🐧 初心者向け Linux コマンド入門

# 🔰 はじめに

Linux（リナックス）は、サーバー運用やプログラミング、研究開発に欠かせないOSです。最初は難しそうに感じるかもしれませんが、基本のコマンドを覚えるだけでグッと自由度が上がります。

---

## **🧬 なぜバイオインフォマティクスにLinuxが必要なの？**

バイオインフォマティクスでは、**DNAやRNAの大量の配列データをコンピュータで処理**します。そのため、**高速・柔軟・自動化しやすい環境**が必要です。ここで登場するのが **Linux** です。

---

### **✅ Linuxが選ばれる理由**

### **1. 🔧 解析ツールのほとんどがLinux対応**

多くの有名なツール（例：BLAST、BWA、GATKなど）は**Linuxでの利用を前提に開発**されています。Windowsでは動かない、または動かしにくいことが多いです。

### **2. 🖥️ コマンド操作で自動化しやすい**

Linuxでは**シェル（bashなど）を使って、コマンドで作業を自動化**できます。何百ものサンプルを一括処理するときにとても便利です。

### **3. 🌐 研究サーバー・クラウド環境でもLinuxが標準**

大学や研究機関のサーバー、クラウド（AWS、Google Cloudなど）でも**Linuxが標準的に使われています**。そのため、Linuxに慣れていないと、環境にアクセスして解析するのが難しくなります。

---

### **🧭 Linuxを完全に避けて通れるか？**

結論から言うと、**Linuxを完全に避けるのは非常に難しい**です。

GUIだけの解析ツール（例：QIAGEN CLCなど）もありますが、

- 対応できる解析が限られる
- 商用で高額な場合がある
- 柔軟な解析やスクリプトによる制御ができない

といった制約があります。

---

### **💡 初心者へのメッセージ**

最初はとっつきにくいかもしれませんが、Linuxは「慣れれば最強の味方」です。

基本的な操作やシェルスクリプトを少しずつ覚えていくことで、**自分の力で解析を進める自由さ**が得られます。

---
# **Linux実習：コマンド・オプション・引数とは**
## **1. コマンドとは**

**コマンド**とは、

👉 コンピュータに「何をしてほしいか」を指示する命令のことです。

### **例**

```bash
ls
```

これは

👉「ファイル一覧を表示せよ」

という命令です。

---

## **2. コマンドの基本構造**

Linux のコマンドは、基本的に以下の形をしています：

```bash
コマンド オプション 引数
```

### **具体例**

```bash
ls -l /home
```

| **要素** | **内容** |
| --- | --- |
| コマンド | ls（一覧表示） |
| オプション | -l（詳細表示） |
| 引数 | /home（対象のディレクトリ） |

---

## **3. 引数（argument）とは**

**引数**とは、

👉「どこに対して操作するか」を指定するものです。

### **例**

```bash
ls /home
```

👉 `/home` ディレクトリの中身を表示する

---

### **よく使う引数の例**

| **コマンド** | **引数の意味** |
| --- | --- |
| `ls dir` | dir の中を見る |
| `cat file.txt` | file.txt の内容を表示 |
| `cd dir` | dir に移動 |

---

## **4. オプション（option）とは**

**オプション**とは、

👉「コマンドの動作を変更する追加設定」です。

多くの場合、`-`（ハイフン）を付けます。

---

### **例**

```bash
ls -l
```

👉 詳細情報（サイズ・日時など）を表示

---

### **複数のオプション**

```bash
ls -la
```

👉 `-l`（詳細表示） + `-a`（隠しファイル表示）

---

### **長いオプション（参考）**

```bash
ls --all
```

👉 `-a` と同じ意味

---

## **5. まとめ（最重要ポイント）**

### **✔ コマンドの意味**

- **コマンド**：何をするか
- **引数**：どこに対して行うか
- **オプション**：どのように行うか

---

### **✔ イメージで理解**

```bash
cp -r dir1 dir2
```

👉 意味：

- `cp`：コピーする（コマンド）
- `r`：フォルダごとコピー（オプション）
- `dir1` → `dir2`：コピー元とコピー先（引数）

---

## **6. よくある初心者のつまずき**

### **❌ スペースを忘れる**

```bash
ls-l   ← 間違い
ls -l  ← 正しい
```

---

### **❌ オプションと引数の順番**

基本的には：

```bash
コマンド → オプション → 引数
```

---

### **❌ ファイルが存在しない**

```bash
cat test.txt
```

👉 ファイルが無いとエラー

---

# 📚 ディレクトリを表す記号（パス記号）の基礎知識

| 記号 | 意味と使い方 |
| --- | --- |
| `~` | ホームディレクトリ（自分の作業場所）を意味します。例：`cd ~` は自分のホームに移動。 |
| `.` | 今いるディレクトリ（カレントディレクトリ）を指します。例：`./script.sh` は、今いる場所にあるスクリプトを実行。 |
| `..` | 一つ上の階層のディレクトリを意味します。例：`cd ..` は親ディレクトリへ移動。 |
| `/` | ルートディレクトリ（ファイルシステムの最上位）。パスの区切りにも使います（例：`/home/user`）。 |

---

## **📁 絶対パスと相対パスってなに？**

ファイルやフォルダ（ディレクトリ）を指定するとき、**「場所（パス）」をどう書くか**には2つの方法があります：

---

### **🛣️ 1. 絶対パス（absolute path）**

👉 **「どこにいても通じる完全な道順」**

- ルートディレクトリ（ / ）から始まるフルパス。
- 例：

```
/home/user/data/sample.txt
```

実際に、以下を実行してみてください

```
pwd

```

現在の作業ディレクトリの絶対パスが以下のように表示されるはずです

```
/workspaces/misa_analysis

```

全体のディレクトリ構造はこんな感じです。

```
/                  # ルート
├── bin -> usr/bin
├── boot
├── dev
├── etc
├── home
├── lib -> usr/lib
├── lib64 -> usr/lib64
├── media
├── mnt
├── opt
├── proc
├── root
├── run
├── sbin -> usr/sbin
├── srv
├── sys
├── tmp
├── usr
├── var
├── vscode
└── workspaces　#　現在の作業ディレクトリは、この階層の下に格納された "misa_analysis" になります

```


🔹 特徴：

- いつでも同じ場所を指す。
- スクリプトの中では安定していて便利。

---

### **🧭 2. 相対パス（relative path）**

👉 **「今いる場所（カレントディレクトリ）からの道順」**

- 現在の場所からの距離を指定。
- 例（現在のディレクトリが /home/user のとき）：

```
./data/sample.txt     # カレントディレクトリ内の data フォルダ
../shared/info.txt    # 1つ上のフォルダから shared/info.txt へ
```

以下の2つの実行結果を比べてみてください

相対パスでディレクトリを指定した場合
```
ls ./チュートリアル

```
ls コマンドは、ディレクトリ内のファイル名を出力するコマンドです。

絶対パスでディレクトリを指定した場合
```
ls /workspaces/misa_analysis/チュートリアル

```

🔹 特徴：

- 相対パスは、作業場所によって指すファイルが変わる。
- よく使うが、場所を間違えると「ファイルが見つかりません」になりやすい。

---

### **🧪 バイオインフォマティクスでのポイント**

- **スクリプトを書くときは「絶対パス」が安全**。
- **ターミナルで移動した直後や一時的な操作では「相対パス」が便利**。

---

### **💡まとめ表**

| **種類** | **書き方** | **メリット** | **デメリット** |
| --- | --- | --- | --- |
| 絶対パス | /から始まるフルパス | いつでも確実に使える | 長くなりがち |
| 相対パス | ./ や ../ を使う | 短く書けて柔軟 | 作業場所に依存して不安定 |

# **初心者が最初に覚えるべきコマンド10選**

## 📁 1. `ls`：ファイルやフォルダの一覧表示

```bash
ls

```
現在の作業ディレクトリ内のファイル、ディレクトリが表示されます

- 今いるディレクトリの中身を表示します
- `ls -l`：詳細表示（サイズ・日付など）
- `ls -a`：隠しファイルも表示（例：`.bashrc`）

特定のディレクトリ内のディレクトリを指定したい場合は、ディレクトリのパスを入力する

```bash
ls ./チュートリアル

```


### **🃏 ワイルドカード（複数ファイルの指定）**

**ワイルドカード**を使うと、**同じ拡張子を持つファイルを一括操作**できます。

**🎯 よく使うワイルドカード一覧**

| **パターン** | **意味** | **例** |
| --- | --- | --- |
| * | 0文字以上のすべての文字列 | *.txt → 全ての .txt ファイル |
| ? | 任意の1文字 | file?.txt → file1.txt など |
| [abc] | a, b, c のどれか1文字 | file[12].txt → file1.txt など |
| [a-z] | a〜z の範囲の1文字 | file[a-c].txt → filea.txt など |


- **📂 例：*.fasta.gz をまとめて表示**

以下を実行してみよう。
```
ls ./fastas/*.fasta.gz

```

100ファイルが出力されてしまいます

今度は、特定の染色体のファイルだけ出力しましょう

```
ls ./fastas/Chr03*.fasta.gz

```
第3染色体のファイルだけ出力されるはずです。

---

## 📂 2. `pwd`：現在の場所を表示

```bash
pwd

```

- 今どのディレクトリにいるか確認できます


---

## 📄 3. `cd`：ディレクトリの移動

```bash
cd [フォルダ名]

```

以下を順番に実行してみよう

```bash

cd ./fastas
pwd
cd ..
pwd

```

- `cd ./fastas`：fastas フォルダに移動
- `cd ..`：一つ上の階層へ移動



---

## 📖 4. `cat`：ファイルの中身を表示

実際に実行してみよう

```bash
ls *.txt
cat sample1.txt
cat sample2.txt

```

- 複数ファイルもまとめて表示可能：`cat sample1.txt sample2.txt`


---
## 🔍 5. **`head` : 長いテキストファイルの冒頭部の閲覧専用コマンド**



```
head ファイル名
```

やってみよう！

```
head  Chr6.tsv
```
冒頭部10行が表示される

冒頭部15行を閲覧したい場合

```
head -15 Chr6.tsv
```

---

## 🔍 6. **`less` : 長いテキストファイルを少しずつスクロールして読むための閲覧専用コマンド**


```
less ファイル名
```

やってみよう！

```
less Chr6.tsv
```

 終了する場合は, q を叩く

---


### **🔸 よく使う操作キー**

| **操作** | **内容** |
| --- | --- |
| ↓ / j | 1行下へスクロール |
| ↑ / k | 1行上へスクロール |
| スペース | 1ページ下へスクロール |
| b | 1ページ上へスクロール |
| /文字列 | 検索（Enterで次の一致へ） |
| n | 次の検索結果へジャンプ |
| q | 閲覧終了（lessを終了） |

 **「/」は「または（or）」の意味**です。

---

### **💡 ポイント**

- cat よりも便利：長いファイルでもスクロールして読める！
- テキストを「編集」することはできません。**読むだけ**。

## 📑 7. `touch`：空ファイルを作る

```bash
touch test.txt

```

- 空のファイル `test.txt` を作成
- 既にあるファイルに対しては「更新日時」を変更

---

## 🗃️ 8. `mkdir`：新しいフォルダを作る

```bash
mkdir my_folder

```

- `my_folder` という名前のフォルダを作成

---

## 🗑️ 9. `rm`：ファイルやフォルダを削除

```bash
rm file.txt

```

- `rm -r folder_name`：フォルダごと削除（再帰的）
⚠️ 削除は元に戻せないので注意！

---

## 📋 10. `cp`：ファイルをコピーする

```bash
cp file.txt copy.txt

```

- `file.txt` を `copy.txt` としてコピー
- `cp -r dir1 dir2`：フォルダをコピー

---

## 🔀 11. `mv`：名前の変更・移動

```bash
mv old.txt new.txt

```

- `old.txt` を `new.txt` にリネーム
- または別の場所に移動

---

---

## **🛑 `exit` : 現在のシェル（またはスクリプト）の実行を終了**

```
exit
```

# 🧠 おすすめの覚え方

- 最初は「`ls` → `cd` → `pwd`」だけでもOK
- 触って慣れるのが一番！仮想環境や WSL でも試せます

# 🛠 便利な小技（おまけ）

- `Tabキー`：コマンドやファイル名の補完
- `↑ / ↓`：履歴をたどって再実行
- `Ctrl + C`：今の操作を強制停止
